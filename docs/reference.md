# Fin-Radar - 金融雷达 技术参考手册

## 一、项目背景

### 1.1 项目简介
- **名称**：Fin-Radar - 金融雷达
- **用途**：财报/公告关键指标解析与风险预警
- **特点**：财报分析纯本地运行 + A 股实时行情查询

### 1.2 技术特点
- **纯本地运行**：财报分析采用规则引擎，无需外部 API
- **隐私安全**：财报数据不离开本地设备
- **行情查询**：支持 A 股实时行情（需联网，数据来源：东方财富网）
- **编程语言**：Python 3.10+

## 二、财务指标说明

### 2.1 核心指标定义

#### 营业收入 (Revenue)
- **定义**：公司在正常经营活动中获得的收入总额
- **计算公式**：主营业务收入 + 其他业务收入
- **单位**：元（或亿元）
- **重要性**：反映公司经营规模和市场占有率

#### 归母净利润 (Net Profit)
- **定义**：归属于母公司股东的净利润
- **计算公式**：净利润 - 少数股东损益
- **单位**：元（或亿元）
- **重要性**：反映公司盈利能力

#### 资产负债率 (Debt Ratio)
- **定义**：总负债与总资产的比率
- **计算公式**：(总负债 / 总资产) × 100%
- **单位**：%
- **重要性**：反映公司偿债能力和财务风险

#### 经营现金流 (Operating Cash Flow)
- **定义**：经营活动产生的现金流量净额
- **计算公式**：经营活动现金流入 - 经营活动现金流出
- **单位**：元（或亿元）
- **重要性**：反映公司现金流健康状况

#### 研发投入占比 (R&D Ratio)
- **定义**：研发费用占营业收入的比例
- **计算公式**：(研发费用 / 营业收入) × 100%
- **单位**：%
- **重要性**：反映公司技术创新投入力度

#### 营收同比增长率 (Revenue YoY)
- **定义**：营业收入与上年同期相比的变化率
- **计算公式**：((本期营收 - 上期营收) / 上期营收) × 100%
- **单位**：%
- **重要性**：反映公司成长性

#### 净利润同比增长率 (Net Profit YoY)
- **定义**：净利润与上年同期相比的变化率
- **计算公式**：((本期净利润 - 上期净利润) / 上期净利润) × 100%
- **单位**：%
- **重要性**：反映公司盈利能力变化趋势

## 三、风险评估标准

### 3.1 负债风险评估

| 风险等级 | 资产负债率 | 说明 |
|---------|-----------|------|
| 高风险 | >70% | 负债水平过高，偿债压力大 |
| 中风险 | 60-70% | 负债水平偏高，需关注 |
| 低风险 | <60% | 负债水平正常 |

**建议措施**：
- 高风险：评估债务违约风险，关注偿债能力
- 中风险：持续关注负债变化趋势
- 低风险：无需特别关注

### 3.2 盈利风险评估

| 风险等级 | 净利润同比变化 | 说明 |
|---------|---------------|------|
| 高风险 | 下滑 >30% | 断崖式下跌，盈利能力严重恶化 |
| 中风险 | 下滑 15-30% | 明显下滑，盈利能力减弱 |
| 低风险 | 下滑 <15% 或增长 | 盈利能力稳定或改善 |

**建议措施**：
- 高风险：深入分析下滑原因，关注主营业务
- 中风险：关注盈利能力变化趋势
- 低风险：持续观察

### 3.3 现金流风险评估

| 风险等级 | 经营现金流 | 说明 |
|---------|-----------|------|
| 高风险 | 负值 | 经营活动现金净流出，资金链风险 |
| 低风险 | 正值 | 经营活动现金净流入，现金流健康 |

**建议措施**：
- 高风险：关注现金流状况和融资能力
- 低风险：现金流状况良好

### 3.4 增长风险评估

| 风险等级 | 营收同比变化 | 说明 |
|---------|-------------|------|
| 高风险 | 下滑 >10% | 业务萎缩，市场竞争力下降 |
| 中风险 | 下滑 0-10% | 增长放缓 |
| 低风险 | 增长 | 业务扩张 |

**建议措施**：
- 高风险：分析行业环境和公司竞争力
- 中风险：关注业务发展趋势
- 低风险：持续观察

### 3.5 研发投入风险评估

| 风险等级 | 研发投入占比 | 说明 |
|---------|-------------|------|
| 中风险 | <3% | 研发投入不足，技术竞争力弱 |
| 低风险 | ≥3% | 研发投入充足 |

**建议措施**：
- 中风险：关注公司技术创新能力
- 低风险：无需特别关注

## 四、技术实现细节

### 4.1 本地规则引擎

**Fin-Radar** 采用纯本地规则引擎处理财报，核心特点：
- ✅ 财报分析无需外部 API 调用
- ✅ 财报数据不离开本地设备
- ✅ 财报功能零网络依赖，离线可用
- ✅ 隐私安全，敏感数据不会泄露

### 4.2 PDF 解析技术

#### pdfplumber
- **优点**：功能强大，支持复杂 PDF 格式，提取精度高
- **缺点**：依赖较重，处理速度稍慢
- **适用场景**：正式生产环境

#### PyPDF2
- **优点**：轻量级，兼容性好，处理速度快
- **缺点**：对复杂 PDF 支持有限
- **适用场景**：快速处理或作为备选方案

### 4.3 正则表达式模式

#### 营业收入提取
```python
r'(?:营业收入|营业总收入|营收)[：:为是]?\s*([^\n]{5,50}(?:元|万元|亿元))'
```

#### 净利润提取
```python
r'(?:归母净利润|归属于母公司股东的净利润)[：:为是]?\s*([^\n]{5,50}(?:元|万元|亿元))'
```

#### 资产负债率提取
```python
r'(?:资产负债率|负债率)[：:为是]?\s*([^\n]{3,20}%?)'
```

### 4.4 数字解析

#### 支持的格式
- 普通数字：123, 45.67
- 带单位数字：52.3亿元, 15.2万亿
- 百分比：12.3%, 15.6％

#### 转换规则
- 亿元 → × 100,000,000
- 万元 → × 10,000
- 百分比 → 数值（去掉 % 号）

## 六、常见问题解答

### Q1: PDF 解析失败怎么办？
**A**: 
1. 检查 PDF 是否为扫描版（图片格式）
2. 尝试使用 OCR 工具预处理
3. 手动提取文本后使用 `--text` 参数

### Q2: 指标提取不准确怎么办？
**A**:
1. 检查输入文本格式是否规范
2. 确保文本包含标准的财务术语
3. 手动修正后重新分析

### Q3: 如何提高风险评估准确性？
**A**:
1. 确保提取的指标数据准确
2. 结合行业特点进行分析
3. 参考多家同行业公司数据
